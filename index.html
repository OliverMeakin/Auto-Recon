What it does

A configurable reconciliation core that matches within groupings, logs outputs, and routes exceptions.

Matches the way accountants think

Exact matches and grouped matches to net to zero within a configurable key (account, vendor, entity, project, cost center, or any custom grouping).

Enforces your rules

Prevent invalid combinations with configurable constraints (type locks, restricted pairings, cardinality rules). Flag uncertain items instead of forcing them through.

Produces an audit-friendly output

Writes statuses back to the working sheet, appends cleared groups to an append-only log, and generates a trace output to support review, transparency, and troubleshooting.

Outputs you can hand to reviewers

  • Status column for quick close workflows
  • Cleared log (append-only) for audit support
  • Review queue for exceptions and weak matches
  • Trace output for “why did this match?” questions

Configurable, not custom-coded

  • Map your columns once (grouping key, amounts, optional memo/source)
  • Set tolerance, matching modes, and guardrails
  • Reuse the same engine across multiple reconciliations
  • Templates available for common patterns

How it works

A straightforward workflow designed for month-end and audit-readiness.

1

Configure

Choose your grouping key, amount columns (net or debit/credit), and optional memo/source fields.

2

Run

The engine matches within each group using your rules. Uncertain items are routed to review.

3

Review and export

Use the cleared log and trace output to support internal review and audit requests.

Excel-native delivery: ships as an Excel workbook “app shell” + automation scripts. Minimal change management and fast adoption for accounting teams.

Use cases

A few common reconciliation patterns where rules-based matching saves time.

Bank and cash activity

Statement vs ledger activity, timing differences, tolerance handling, and exception review routing.

Clearing accounts

High-volume clearing activity where grouped matching and guardrails reduce manual work.

Intercompany and settlements

Entity-based groupings, many-to-many settlement matching, and standardized trace output.

Custom template support

If your export format is unique, we can build a tailored configuration and matching rules around your structure. Sanitized sample data is fine.
