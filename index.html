What it does

Finds matches you’d do manually

Exact pairs, subset matches (many-to-one / many-to-many), and tolerance-based matches (ex: ±$0.01) inside a defined bucket.

Enforces your internal rules

Lock matching by type, restrict source-to-source pairings, and prevent “bad matches” automatically while still surfacing exceptions for review.

Produces an audit trail

Writes statuses back to your working sheet, appends cleared groups to an append-only log, and generates a debug trace for troubleshooting.

How it works

1) Configure your rules

  • Pick your “bucket” field (account, vendor, entity, project, etc.)
  • Select amount columns (debit/credit/net) and optional memo/source fields
  • Set match types: exact, subset, tolerance, spillover (optional)

2) Run the engine

  • Matches within each bucket to net to zero
  • Applies constraints (type locks, source restrictions, cardinality rules)
  • Flags “Needs Review” instead of forcing weak matches

3) Review outputs

  • Status column updates (Matched, Cleared, Tolerance Flag, Needs Review)
  • Cleared sheet: append-only audit log
  • Debug/trace sheet for transparency
Excel-native delivery: Ships as an Excel workbook + Office Script “app shell.” No new systems, minimal friction, and accounting teams can own the workflow.
