What it does

Finds real matches

Pairs and subset-matches transactions that net to zero inside a Vess/Voy bucket, with optional spillover checks.

Enforces your rules

SS can match many-to-one-or-many. MED enforces exactly one offshore line. ADP↔ADP blocked. SS and MED never mix.

Produces an audit trail

Writes Status back to the Recon sheet, appends cleared groups to a Cleared sheet, and creates a debug trace for review.

How it works

Role-aware matching

  • ADP PAYMT lines match against OFFSHORE sources (KPS/MGA/EIB)
  • Never allow ADP to match ADP
  • Exceptions: “MEBA PRB” or “MMP” can match broadly (still type-locked)

Type constraints

  • FICA SS and FICA MED are locked and never mix
  • SS: many ADP payments can clear to one or more offshore offsets
  • MED: many ADP payments must clear to exactly one offshore offset

Outputs

  • Status column updates (Matched, Cleared, .01 variance, Needs Review)
  • Cleared sheet append-only log
  • Recon_Debug trace for troubleshooting
Implementation: This is designed to ship as an Excel workbook + Office Script “app shell”. It looks like software, but stays inside the environment your accounting team already uses.
